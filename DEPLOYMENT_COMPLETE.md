## 📋 Your Next 3 Steps (2 Hours Total)

### Step 1: Set Up Database (20 minutes)
```
1. Go to https://vercel.com/dashboard
2. Click your "trendtok-dashboard" project → Storage
3. Create Postgres database (name: "trendtok-db")
4. Copy connection string
5. Add DATABASE_URL to Vercel Environment Variables
6. Load schema: Copy setup-database.sql to Vercel SQL editor → Run
```

### Step 2: Create Dify Workflows (45 minutes)
```
1. Go to https://dify.ai and sign in (free)
2. Create 4 workflows:
   - trending-discovery
   - video-generation
   - tiktok-posting
   - performance-tracking
3. Get your Dify API key
4. Add to Vercel Environment Variables:
   - VITE_DIFY_API_KEY
   - VITE_DIFY_API_URL=https://api.dify.ai/v1
```

**See DATABASE_AND_DIFY_SETUP.md for detailed instructions**

### Step 3: Add API Keys (15 minutes)
```
Get keys from:
1. Google Cloud Console → Google Veo API
2. OpenAI → API Keys
3. TikTok Developer Portal
4. Use your existing TinyFish key

Add all to Vercel Environment Variables
(See .env.production.example for complete list)
```

## 🐛 If Something Goes Wrong

### Dashboard doesn't load
- Check browser console (F12)
- Verify Vercel deployment is "Ready"
- Check no build errors in Vercel logs

### Dify workflows fail
- Check Dify execution logs in Dify Studio
- Verify API keys are correct
- Test each HTTP node individually
- Check database connection in Dify

### No data appears in dashboard
- Verify DATABASE_URL is set in Vercel
- Check VITE_DIFY_API_KEY is correct
- Verify workflows are published in Dify
- Monitor browser Network tab for API errors

**See DATABASE_AND_DIFY_SETUP.md "Troubleshooting" for detailed help**
